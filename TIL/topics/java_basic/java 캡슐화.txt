java 캡슐화

0. 필드와 메서드를 하나의 캡슐의 형태로 묶어서 데이터를 외부로부터 보호하기 위한 방식
직접적인 접근을 막고 객체를 통한 접근만 허용(참조를 통한 데이터 수정만 가능하기에 변수조작이 힘듬)

1. 접근 제어자

public(+) : 접근제한 없음
protected(#) : 동일패키지, 다른패키지 하위 클래스에서 접근가능
default(~) : 기본값 - 동일 패키기내부에서 접근가능
private(-) : 동일 클래스안에서만 접근 가능

2. get/set

getter : 설정한 변수값 읽기용
setter : 외부에서 메서드에 접근하여 데이터값 변경용

다른 패키지에서 정보를 탐색할때 get인스턴스명/set인스턴스명 메서드를 사용

3. 캡슐화할 클래스에서 전역변수를 private화 하여 변수 제어

4. static final 인스턴스
static 변수 : 별도 메모리에 할당되어 선언한 순간 바로 메서드에 활용 가능한 변수
final : 한번 지정하고 다시 변경하지 않기 위해 설정
>>고정값들을 별도로 저장하기 위해 static final을 달아둠
